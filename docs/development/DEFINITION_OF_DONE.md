# Definition of Done

A feature is considered complete only if:

## Quality

- [ ] Builds successfully
- [ ] Tests pass
- [ ] Lint passes
- [ ] No TypeScript errors

## Architecture

- [ ] Complies with DOMAIN_MODEL.md
- [ ] Complies with PRINCIPLES.md
- [ ] No ADR conflicts

## Product

- [ ] UX reviewed
- [ ] Error states handled
- [ ] Loading states handled
- [ ] Empty states handled

## Runtime

- [ ] Runtime safe
- [ ] History preserved
- [ ] Finished matches unchanged

## Database

- [ ] Migration reviewed
- [ ] RLS considered
- [ ] Group isolation preserved

## Documentation

- [ ] Docs updated
- [ ] Release checklist updated if required

Only after every item is complete may the feature be considered Done.
